---
"@ima/plugin-testing-integration": major
---

`@ima/plugin-testing-integration` now delegates JSDOM and environment setup to `@ima/testing-library`, removing the built-in server-render and JSDOM initialization logic.

- **What** `initImaApp` no longer creates a JSDOM instance, starts an IMA server to generate HTML, or builds a locale dictionary from language files. Instead it validates that a JSDOM environment is already present (provided by `@ima/testing-library/jest-preset`), applies the configured `environment` value to `window.$IMA.$Env`, installs an internal fake dictionary proxy for localization, and boots the IMA app via `@ima/core`. `clearImaApp` now also resets the internal timer collection and restores `console.assert` between test runs. The configuration options `masterElementId`, `protocol`, `host`, `locale`, `beforeCreateIMAServer`, `afterCreateIMAServer`, `pageScriptEvalFn`, `processEnvironment`, and `applicationFolder` have been removed. The source modules `helpers.js` and `localization.js` have been deleted. The direct dependencies `jsdom`, `@messageformat/core`, and `globby` have been removed, and the `@ima/cli` peer dependency has been removed. `@ima/testing-library` and `@testing-library/react` are now peer dependencies. The `EnzymePageRenderer.js` file has been deleted; `enzyme` and `react-dom` are newly added as peer dependencies for consumers who provide a custom `TestPageRenderer` that requires them. A package `exports` map has been added, defining explicit ESM/CJS entry points.
- **Why** With the introduction of `@ima/testing-library`, JSDOM setup, server configuration, and dictionary generation were consolidated into a shared package. Relying on it avoids duplicated infrastructure, ensures consistency across projects, and lets consumers use the React Testing Library utilities (`renderWithContext`, `screen`, `waitFor`, etc.) that `@ima/testing-library` already provides.
- **How** Install `@ima/testing-library` and `@testing-library/react` as dev dependencies. Set `preset: '@ima/testing-library/jest-preset'` in your jest configuration. Remove any manual JSDOM initialisation from your tests. Remove any `setConfig` calls that used the removed options listed in the **What** bullet above. If you were using `EnzymePageRenderer`, replace it with `@ima/testing-library`'s `renderWithContext` and related utilities.
